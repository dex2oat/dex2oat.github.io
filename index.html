v2raya多节点分流

post @ 2024-07-24

因GooglePlay区分地区需要不同代理节点

例如需要5个不同的IP组，名为HK，JP，US，ID，BR

1. 新增站点

在v2raya的上方新增5个出站，命名为HK，JP，US，ID，BR。

2. 选择节点

依次选择出站，在每个出站的界面中连接特定的服务器节点。

3.设置模式

将“规则端口的分流模式”选为“RoutingA”。

4. 写入规则

在代码中使用30001-30005端口以选择不同的IP出口。

matrix

post @ 2024-07-23

Matrix 是一个开源、可交互、去中心化的实时通信服务框架。使用 Matrix 可以搭建安全的通信服务器，配合支持 Matrix 的客户端可以实现个人、团队间的实时聊天交互，对Matrix Dendrite搭建聊天服务器的过程感兴趣的朋友一起看看吧

部署方式

docker-compose

version: "3"
services:
  db:
    restart: always
    image: postgres:latest
    shm_size: 1024mb
    container_name: synapse-postgres
    networks:
      - dbs_network
    healthcheck:
      test: ["CMD", "pg_isready", "-U", "postgres"]
    volumes:
      - ./postgres:/var/lib/postgresql/data
    environment:
      - "PUID=1030"
      - "PGID=100"
      - "TZ:=Asia/Shanghai"

  redis:
    restart: always
    image: redis:latest
    container_name: synapse-redis
    networks:
      - dbs_network
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
    volumes:
      - ./redis:/data
    environment:
      - "PUID=1030"
      - "PGID=100"
      - "TZ:=Asia/Shanghai"

  synapse:
    image: matrixdotorg/synapse:latest
    container_name: matrix-synapse
    ports:
      - 38008:8008
    depends_on:
      - db
      - redis
    volumes:
      - ./data:/data
    restart: always
    networks:
      - synapse_network
      - dbs_network
    environment:
      - "PUID=1030"
      - "PGID=100"
      - "TZ:=Asia/Shanghai"
networks:
  synapse_network:
  dbs_network:
    internal: true

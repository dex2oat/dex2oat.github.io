宁波电信换VSOL2802RH光猫

post @ 2024-07-24

设备参数

技术 描述
PON 接口 1 G/EPON port(EPON PX20+ and GPON Class B+) 收光灵敏度: ≤-28dBm 发光功率: 0~+4dBm 传输距离: 20KM
波长 Tx1310nm,Rx 1490nm
光接口 SC/PC连接头
接口 1* 10/100/1000Mbps and 1* 10/100/1000/2500Mbps 自适应以太网接口。 全/半双工, RJ45 连接头。
指示灯 5 指示灯, SYS, PON, LOS, LAN1~2
运行温度 -5℃～55℃，10%～90%(非凝结)
存储条件 -30℃～60℃，10%～90%(非凝结)
供电 DC 12V, 0.5A
能耗 ≤4W
尺寸 100mm92mm29.5mm(长宽高)
净重 0.11Kg

接口/按钮面板

名称 功能
PON 通过SC型光纤连接器，单模光纤电缆连接到OLT。
LAN1/2 通过Cat5e电缆，RJ-45连接器，通过以太网端口连接PC或其他设备。
RST 按RST按钮超过10秒，ONU恢复工厂默认值并重新启动。
DC 12V 连接电源适配器。DC 12V, 0.5A。

指示灯面板

指示灯 标志 状态 描述
系统 SYS 常亮/熄灭 系统未运行或出现致命错误。
闪烁 正常运行。
注册 PON 常亮 设备已注册到PON系统。
熄灭 设备未注册到PON系统。
闪烁 设备正在注册。
光信号 LOS 闪烁 设备未接收到光信号。
熄灭 设备接收到光信号。
局域网 LAN1 LAN2 常亮 端口连接正确。
熄灭 端口连接异常或未连接。
闪烁 端口正在发送或/和接收数据。

设置

在开始前需要知道原本光猫的如下信息：

  • 宽带的用户名及密码
  • VLAN ID
  • LOID
  • SN 码 （有些地区需要）
  • MAC 地址 （有些地区需要）

宁波电信

VLAN ID上网为41
IPTV为43(仅供参考，不同地区有差异)

一、添加WAN口信息

设置–> 广域网–> 广域网

信道模式为Bridge
VLAN ID: 41
VLAN Cos: 2 (默认为0)
LAN: LAN2 (2.5G口)

二、设置LOID及认证ID

设置–> 广域网–> GPON设置

宁波电信到这一步应该可以正常拨号上网了。

不能拨号的尝试修改SN和MAC，这里不展开了。

v2raya多节点分流

post @ 2024-07-24

因GooglePlay区分地区需要不同代理节点

例如需要5个不同的IP组，名为HK，JP，US，ID，BR

1. 新增站点

在v2raya的上方新增5个出站，命名为HK，JP，US，ID，BR。

2. 选择节点

依次选择出站，在每个出站的界面中连接特定的服务器节点。

3.设置模式

将“规则端口的分流模式”选为“RoutingA”。

4. 写入规则

在代码中使用30001-30005端口以选择不同的IP出口。

matrix

post @ 2024-07-23

Matrix 是一个开源、可交互、去中心化的实时通信服务框架。使用 Matrix 可以搭建安全的通信服务器，配合支持 Matrix 的客户端可以实现个人、团队间的实时聊天交互，对Matrix Dendrite搭建聊天服务器的过程感兴趣的朋友一起看看吧

部署方式

docker-compose

version: "3"
services:
  db:
    restart: always
    image: postgres:latest
    shm_size: 1024mb
    container_name: synapse-postgres
    networks:
      - dbs_network
    healthcheck:
      test: ["CMD", "pg_isready", "-U", "postgres"]
    volumes:
      - ./postgres:/var/lib/postgresql/data
    environment:
      - "PUID=1030"
      - "PGID=100"
      - "TZ:=Asia/Shanghai"

  redis:
    restart: always
    image: redis:latest
    container_name: synapse-redis
    networks:
      - dbs_network
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
    volumes:
      - ./redis:/data
    environment:
      - "PUID=1030"
      - "PGID=100"
      - "TZ:=Asia/Shanghai"

  synapse:
    image: matrixdotorg/synapse:latest
    container_name: matrix-synapse
    ports:
      - 38008:8008
    depends_on:
      - db
      - redis
    volumes:
      - ./data:/data
    restart: always
    networks:
      - synapse_network
      - dbs_network
    environment:
      - "PUID=1030"
      - "PGID=100"
      - "TZ:=Asia/Shanghai"
networks:
  synapse_network:
  dbs_network:
    internal: true
